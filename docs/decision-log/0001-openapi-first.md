# 0001 OpenAPI-first Architecture

Date: 2026-04-02

We will treat the official TP-Link Omada Open API as the default integration contract.

Reasons:

- it is the vendor-supported path
- it maps cleanly to typed client wrappers
- it is easier to document and test

Sources:

- [Omada Open API docs](https://omada-northbound-docs.tplinkcloud.com/#/home)
- [OpenAPI setup guide](https://community.tp-link.com/en/business/kb/detail/412930)
